Features Overview

Login

Local Login

Users can create an account using their first name, last name, email, and a password.

Existing users can sign in with their email and password.

For convenience, users can check the "Remember me" box to automatically log in for 24 hours.

OAuth2 Login

Users can conveniently log in using their existing accounts on popular platforms like Google, Facebook, or GitHub.

Only the user's Unique ID, Username, and Email will be stored in the database of this app.

If the email is unavailable, a placeholder email will be used with the user's first name, which the user can update later.

If the username is unavailable, a username based on the OAuth2 provider and ID will be generated.

Guest Login

Guests may test out the app by signing in as a guest. This allows them to explore features and functionality without creating an account.

Guest users are deleted after 30 minutes of use to prevent abuse and ensure a fair experience for all users.

Project Management

Project

Users can create and manage projects, organizing their tasks and collaborating with others.

Each project includes a title, description, a list of tasks, and the ability to add collaborators.

Users can mark projects as complete once all tasks are finished.

Projects provide a structured framework for managing complex tasks and achieving goals.

Users can break down large projects into smaller, manageable tasks, track progress, and collaborate effectively with team members.

Collaborators

Project owners can invite existing users to collaborate on their projects.

Only project owners have the ability to add or remove collaborators.

Access to projects is restricted to the project owner and their invited collaborators.

Admins have access to all projects, while other users can only view projects they are involved in.

Tasks

Project owners and Collaborators may create tasks for their projects.

Each task has its own name, description, priority, state, optional deadline, and assigned users.

The tasks in the task list are sorted by priority. (Urgent > High > Medium > Low > Trivial)

A task has one of four states. (New, In Progress, Under Review, Completed)

Tasks may not have deadlines before the current time.

Project owners may assign existing collaborators to tasks.

Comments

Leaving Comments

Users may leave comments in tasks they have been assigned to and edit or delete them.

Users may leave comments starting with '@' and another existing user's username to tag them.

For example, '@hansonsks', '@User Doe', '@Hanson Siu 123'.

Notifications will be sent to tagged users only if they are assigned to the task the comment is in.

Notifications

Receiving Notifications

Users receive notifications for various events in the app, displayed on the right sidebar.

Notifications include updates on tasks, reminders for tasks past their deadline, being added as a collaborator, being assigned to a task, and being tagged in comments.

Clicking the "Mark as Read" button on a notification will mark it as read and remove it from the sidebar, keeping your notifications organized and focused.

User Roles

Admins & Users

The project management app currently has two roles: 1. User 2. Admin

A User can create, edit, and manage their own projects, collaborate with others, and track progress.

An Admin has all the capabilities of a User, plus additional administrative privileges.

This includes managing users, creating and deleting projects, and controlling access permissions.

Security

Role-based Access Control

The app implements role-based access control (RBAC) to ensure that users only have access to the features and data they are authorized to see and modify.

RBAC assigns specific permissions to each role, such as "User" or "Admin," limiting their actions based on their assigned role.

This helps maintain data security, prevent unauthorized access, and ensure that only authorized individuals can perform critical operations.

Unauthorised users may only access the about, javadocs, images, user creation, guest login, and standard login pages.

Database

MySQL/PostgreSQL

The project management app utilizes either MySQL or PostgreSQL as its database management system (DBMS), depending on the chosen cloud service provider.

On cloud platforms like Amazon RDS and Azure SQL Server, both MySQL and PostgreSQL are supported and offer comparable performance.

For deployments on Heroku, PostgreSQL is the recommended choice. It's included as a standard service, while MySQL is available as an add-on with additional costs.

OAuth2 User functions

Github / Google / Facebook

GitHub: Users may conveniently preview their repositories, streamlining access to their projects on GitHub.

Google: Users can quickly access their Google Drive and Google Calendar. (TBD: Integration with both services.)

Facebook: Users can easily navigate to their Facebook homepage and Events to schedule project deadlines.
